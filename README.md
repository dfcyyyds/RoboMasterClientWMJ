# RoboMaster 客户端 (WMJ 定制版)

本项目是一个用于机甲大师（RoboMaster）比赛的跨平台客户端，基于 Unity 开发。它不仅支持连接官方比赛服务器的**比赛模式**，还内置了一个功能完善的**仿真模式**，允许在本地离线进行开发和调试，无需依赖物理机器人或远程服务器。

## 主要功能

- **双模式运行**:
  - **比赛模式**: 连接到官方裁判系统，接收和发送比赛数据。
  - **仿真模式**: 在本地启动一个完整的模拟环境，包括 MQTT 代理和模拟服务器，用于开发和调试。
- **实时图传显示**:
  - 支持接收通过 UDP 传输的 H.264/H.265 视频流。
  - 在 Linux 平台上利用 NVIDIA GPU 的 **NVDEC** 技术进行硬件加速解码，显著降低 CPU 占用率和延迟。
- **数据通信**:
  - 通过 **MQTT** 协议与服务器进行实时、可靠的数据交换。
- **自动化进程管理**:
  - 在仿真模式下，客户端会自动管理 `mosquitto` (MQTT Broker) 和 `mock_server` (模拟器) 进程的生命周期，包括启动、日志记录和关闭。
- **热更新框架**:
  - 支持在运行时动态更新 C# 逻辑脚本，无需重新构建整个客户端，加快开发迭代速度。
- **UI 系统**:
  - 包含启动模式选择、兵种选择、实时比赛数据显示等核心 UI 界面。

## 项目亮点

- **一键启动仿真**: 无需手动配置或启动任何外部程序。只需在启动界面选择“仿真模式”，客户端即可自动在后台搭建并运行完整的本地服务器环境。
- **高性能原生解码**: 深度集成了基于 NVIDIA NVDEC 的原生视频解码插件，实现了 GPU 硬解，相比于纯软件解码，在保证高清画质的同时，极大地提升了性能和响应速度。
- **健壮的跨进程管理**: C# 代码通过精心设计的启动脚本 (`run_process.sh`) 和进程管理逻辑，可靠地控制着外部原生进程 (`mosquitto`, `mock_server`)，并能捕获其输出用于调试，解决了在 Unity 中启动外部进程常见的环境和权限问题。
- **高度集成与自动化**: 从模式选择、进程启动、视频文件分发到网络连接，整个仿真流程高度自动化，为开发者提供了流畅的“开箱即用”体验。
- **清晰的项目架构**: 代码结构清晰，分为框架 (Framework)、功能模块 (Module)、原生插件 (Plugins) 等，便于维护和扩展。

## 部署与使用

### 1. 环境要求

- **操作系统**: 推荐 Ubuntu 22.04+ (主要开发和测试环境)
- **Unity 版本**: `2022.3.x` 或更高版本
- **硬件**:
  - 一块支持 NVDEC 的 NVIDIA 显卡（用于硬件解码）
  - 对应的 NVIDIA 闭源驱动
- **软件依赖 (Linux)**:
  - `mosquitto`: MQTT 代理服务。通过以下命令安装：
    ```bash
    sudo apt update
    sudo apt install mosquitto mosquitto-clients
    ```
  - `ffmpeg`: `mock_server` 依赖 `ffmpeg` 库进行视频编码。
    ```bash
    sudo apt install ffmpeg
    ```

### 2. 编译原生代码

本项目的模拟服务器 `mock_server` 是一个 C++ 程序，需要手动编译。

```bash
cd MockServerCpp/build
cmake ..
make
```
编译成功后，可执行文件将位于 `MockServerCpp/build/mock_server`。

### 3. 运行客户端

#### 在 Unity 编辑器中运行

1.  使用 Unity Editor 打开项目。
2.  打开主场景 `Assets/Scenes/MainScene.unity`。
3.  点击 **Play** 按钮运行。
4.  程序启动后，会首先显示 **模式选择面板**。

#### 使用方法

- **选择仿真模式**:
  1.  在启动面板上，点击 **“仿真模式”** 按钮。
  2.  客户端会自动在后台执行以下操作：
      - 启动 `mosquitto` 进程作为 MQTT Broker。
      - 将 `MockServerCpp/` 目录下的 `.avi` 视频文件复制到 `MockServerCpp/run/` 目录。
      - 启动 `mock_server` 进程。该进程会连接到 `mosquitto`，加载视频文件，并开始向本地 `3334` 端口循环推流。
  3.  客户端主程序会自动连接到本地的 `mosquitto` 和 UDP 端口。
  4.  稍等片刻，您应该能在主界面的图传区域看到视频画面。

- **选择比赛模式**:
  1.  在启动面板上，点击 **“比赛模式”** 按钮。
  2.  客户端将跳过本地进程的启动，并尝试连接到您在 `GameParamsConfig` 中配置的远程裁判系统地址。

### 4. 日志与调试

- **Unity 客户端日志**: 位于项目根目录下的 `Log/DebugLog.txt`。这是最主要的日志文件。
- **原生进程日志**:
  - 在仿真模式下，`mosquitto` 和 `mock_server` 的所有输出（stdout 和 stderr）都会被重定向到 `Log/DebugLog.txt` 文件中，并带有 `[mosquitto]` 或 `[mock_server]` 前缀，方便统一排查问题。
  - 独立的日志文件位于 `MockServerCpp/run/` 目录下 (`mosquitto.log`, `mock_server.log`)。
