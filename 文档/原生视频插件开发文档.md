# 原生视频插件开发计划

## 概述

Native NVDEC + OpenGL 4.5 集成计划，实现 Ubuntu 24.04 + NVIDIA GPU 环境下的零拷贝视频解码路径。

---

## 目标

- 使用 NVDEC (Video Codec SDK) 通过 CUDA 固定功能单元解码 HEVC/H264 AnnexB 码流
- 零拷贝路径：UDP → 原生接收/组帧 → NVDEC 解码 (NV12) → CUDA 颜色转换 → PBO/GL 纹理 → Unity 通过原生插件事件采样纹理 ID
- 保留 C# ffmpeg 管道作为回退方案

---

## 架构设计

### 1. UDP 接收层（原生）

- 使用 `recvmsg` 在 AF_INET/AF_INET6 上接收
- 固定池缓冲区（C++ 预分配 slab，等效于 C# ArrayPool）
- 分片头格式：
  - frameId (u16 LE)：帧编号
  - sliceId (u16 LE)：分片编号
  - frameLen (u32 LE)：帧总长度
  - payload：AnnexB 分片数据
- 帧环形缓冲区按 frameId 索引
- 每帧有序分片映射
- 超时与溢出驱逐
- 丢弃旧帧策略保持低延迟
- 参数集缓存（VPS/SPS/PPS），IDR 到达时在解码前重发参数集

### 2. 解码层（NVDEC/CUVID）

- 创建 CUcontext + CUvideoctxlock + CUvideoparser + CUvideodecoder
- 解析器回调：
  - on sequence → 初始化/重新初始化解码器
  - on picture → 入队解码
  - on display → 映射解码后表面
- 优先支持 HEVC；如检测到则允许 H264

### 3. 颜色转换与上传

- 映射 NVDEC 输出（NV12 格式）
- 使用 CUDA kernel 或 NPP 执行 NV12 → RGB8 转换
- CUDA-OpenGL 互操作：
  - 用 CUDA 注册 PBO/纹理
  - 将 RGB 写入 PBO
  - glTexSubImage 更新常驻纹理
- 双/三缓冲
- 存储最新纹理 ID 供 Unity 获取

### 4. Unity 桥接层

**C 导出接口：**
```c
int nvp_init(int width, int height);
void nvp_push_udp(const uint8_t* ptr, int len);
int nvp_get_latest_texture();
void nvp_shutdown();
```

**C# 封装（NativeVideoBridge）：**
- P/Invoke 调用导出函数
- 使用 `GL.IssuePluginEvent` 或 `CustomTextureUpdateV2` 绑定最新纹理
- 插件加载失败时回退到 ffmpeg 管道

### 5. 日志与诊断

- 高频路径仅保留计数器
- 每秒聚合日志输出
- 可选 verbose 标志用于转储解析器/解码器状态快照

## 构建说明

### 依赖

- CUDA Toolkit（含 Video Codec SDK 头文件）
- OpenGL 开发头文件
- CMake ≥ 3.22

### 目标产物

- `libNativeVideoPlugin.so` 共享库
- 安装位置：`Assets/Plugins/x86_64/`

### CMake 配置要点

```cmake
find_package(CUDA REQUIRED)
find_package(OpenGL REQUIRED)

target_link_libraries(NativeVideoPlugin
    ${CUDA_LIBRARIES}
    nvcuvid
    ${OPENGL_LIBRARIES}
    dl
    pthread
)
```

## 性能预期

| 延迟阶段   | FfmpegPipe+CUDA | NativeNvdec   |
| ---------- | --------------- | ------------- |
| UDP 组帧   | 50-80ms         | 50-80ms       |
| 解码       | 50-80ms         | 20-40ms       |
| 纹理上传   | 20-30ms         | 5-10ms        |
| **总延迟** | **~120-190ms**  | **~75-130ms** |

### Intel 集显兼容性

在 Intel 集显环境下，NVDEC/CUDA 不可用，系统自动降级：

1. HardwareCapabilityDetector 检测到无 NVIDIA GPU
2. VideoStreamService 跳过 NativeNvdec，使用 FfmpegPipe 后端
3. FfmpegPipeDecoder 自动选择 VAAPI 硬件加速（Intel iHD 驱动）
4. 测试结果：Intel ARL 集显环境下全功能正常运行（详见 [Intel集显低配测试报告.md](Intel集显低配测试报告.md)）
