# 图传故障排查指南

## 概述

本文档提供 RoboMaster 客户端 UDP 视频流传输的完整诊断流程，帮助快速定位视频不显示、卡顿、数据丢失等问题。

---

## 架构流程

```
MockServer
  ↓ FFmpeg 编码
  ├─ [VideoSender] 📡 开始推送 HEVC AnnexB
  ├─ [VideoSender] ✅ FFmpeg 进程就绪
  ├─ [VideoSender] 🔍 首次读取: xxx 字节
  └─ [VideoSender] 📊 帧 #N 已发送: M 分片
       ↓ UDP 127.0.0.1:3334
    Unity 客户端
      ↓ UdpClientService (监听 0.0.0.0:3334)
      ├─ [UdpClientService] 🔗 已绑定
      └─ [UdpClientService] 📊 诊断: 包 X, 字节 Y
           ↓ UdpVideoHandler
           ├─ [UdpVideoHandler] 📈 诊断: N 帧, M 切片
           └─ 上抛 UdpVideoFrame
                ↓ AnnexBFrameAssembler
                └─ [组帧完成并上抛]
                     ↓ VideoStreamService
                     ├─ [推送至解码]
                     └─ FfmpegPipeDecoder
                        ├─ [解码帧: WxH]
                        └─ Texture2D.Apply()
                             ↓
                          RawImage 显示
```

## 分阶段诊断

### 阶段 A：UDP 包是否到达？

**检查方法**
```bash
sudo tcpdump -i lo udp port 3334 -vv -c 10
```

**预期输出**
```
10:30:45.123456 127.0.0.1.12345 > 127.0.0.1.3334: UDP, length 1408
```

**故障排查**
| 现象 | 原因 | 解决方案 |
|-----|------|---------|
| 无包输出 | MockServer 未发送 | 检查 `run/mock_server.log` |
| 无包输出 | 防火墙阻断 | `sudo ufw allow 3334/udp` |
| 无包输出 | IP/端口错误 | 检查 `start_mock.sh` 参数 |

### 阶段 B：UDP 包是否被正确解析？

**检查 Console 日志**
```
[UdpClientService] 📊 诊断: 包 X, 字节 Y, 来自 Z
[UdpVideoHandler] 📈 诊断: N 帧, M 切片抵达
```

**故障排查**
| 现象 | 原因 | 解决方案 |
|-----|------|---------|
| 包=0 | 未调用 StartReceive() | 检查 NetworkManager 初始化 |
| 切片=0 | 分片解析失败 | 检查 UdpVideoHandler 逻辑 |

### 阶段 C：组帧是否成功？

**检查日志**
```
[VideoStreamService] 每秒统计 slices=X, assembled=Y, decoded=Z, applied=W
```

**故障排查**
| 现象 | 原因 | 解决方案 |
|-----|------|---------|
| assembled=0 | 超时太激进 | 增加 timeoutSec (0.1→0.2) |
| assembled=0 | 分片丢失 | 检查网络/缓冲区大小 |
| assembled < slices/N | 部分帧超时 | 增加 maxBufferedFrames |

### 阶段 D：解码是否成功？

**检查日志**
```
[FfmpegPipeDecoder] 解码帧: 1280x720
```

**故障排查**
| 现象 | 原因 | 解决方案 |
|-----|------|---------|
| decoded=0 | 参数集丢失 | 确保 IDR 前重发 VPS/SPS/PPS |
| decoded=0 | 编码格式不匹配 | 检查 Codec 自动检测 |
| FFmpeg 错误 | POC/NAL 解析失败 | 检查码流完整性 |

### 阶段 E：纹理是否应用？

**检查日志**
```
[VideoStreamService] 新建/调整纹理: 1280x720
```

**故障排查**
| 现象 | 原因 | 解决方案 |
|-----|------|---------|
| applied=0 | IDR 门控未解除 | 等待或调整 gateTimeoutSec |
| applied≈1 | 限频过严 | 增加 maxApplyFps |

## 常见问题速查

### 问题 1：视频完全黑屏

**排查流程**
1. 检查 `slices` 计数是否 > 0
2. 检查 `assembled` 计数是否 > 0
3. 检查 `decoded` 计数是否 > 0
4. 检查 FFmpeg stderr 是否有错误

### 问题 2：视频卡顿/帧回退

**可能原因**
- GC 压力过大 → 参见 GC 优化架构文档
- 解码队列积压 → 增加 maxDrainPerUpdate
- 组帧超时不当 → 调整 timeoutSec

### 问题 3：NativeNvdec 初始化失败

**错误**
```
[WARN] Native NVDEC init 失败，回退 Ffmpeg (code=-1)
```

**原因**：在主线程调用时 OpenGL 上下文未就绪

**当前方案**：使用 FfmpegPipe + CUDA 硬件加速作为替代

### 问题 4：IPv6 双栈 Socket 异常

**错误**
```
The AddressFamily InterNetworkV6 is not valid for the System.Net.IPEndPoint
```

**解决**
```csharp
EndPoint remote = socket.AddressFamily == AddressFamily.InterNetworkV6
    ? new IPEndPoint(IPAddress.IPv6Any, 0)
    : new IPEndPoint(IPAddress.Any, 0);
```

### 问题 5：重复包处理

**现象**：每个 UDP 包被处理 2 次

**原因**：同时触发了两个事件

**解决**：改为互斥分发
```csharp
if (OnMessageReceivedSegment != null) {
    OnMessageReceivedSegment.Invoke(...);
} else if (OnMessageReceived != null) {
    OnMessageReceived.Invoke(...);
}
```

## 诊断工具

### 1. tcpdump 监听

```bash
sudo tcpdump -i lo udp port 3334 -vv -c 10
```

### 2. 端口占用检查

```bash
ss -tuln | grep 3334
```

### 3. 防火墙状态

```bash
sudo ufw status
```

### 4. 服务器日志

```bash
tail -f Assets/MockServerCpp/run/mock_server.log
```

### 5. 客户端日志

```bash
tail -f Log/RunLog.txt | grep -E "slices|assembled|decoded|applied"
```
