一键安装

复制即用 — 首次安装
Linux（bash / zsh）默认装到 ~/RoboMasterClient
mkdir -p ~/RoboMasterClient && cd ~/RoboMasterClient && \
curl -sSL https://antientropy.xin/robomaster/Linux64/launch.sh -o launch.sh && \
chmod +x launch.sh && ./launch.sh
下次启动 在同一目录运行 ./launch.sh。如有更新，客户端会在游戏内弹出带进度条的更新面板。
Windows（cmd / PowerShell）默认装到 C:\RoboMasterClient
mkdir C:\RoboMasterClient & cd /d C:\RoboMasterClient & ^
curl -o launch.bat https://antientropy.xin/robomaster/Windows64/launch.bat & ^
launch.bat
下次启动 双击 launch.bat。客户端启动后会自动弹出更新提示。

自动增量更新机制

两层保障——启动器层面 / 客户端层面，均基于 manifest.json + SHA-256 短路，避免重复下载与重复拷贝。

游戏内自动检测（推荐）
  • 客户端启动时立刻对比远端 manifest 与本地 manifest.local.json
  • 检测到更新 → 弹出与 SettingsPanel 同样风格的更新面板（含版本对比、文件数、下载体积）
  • 用户按 [U] 立即更新 → 切换到下载进度页：总进度条 / 当前文件 / 下载速率
  • 下载到 .update_staging/ 暂存目录，每个文件 SHA-256 校验通过后保留
  • 全部完成 → 写 swap 脚本 → Unity 退出 → 脚本原子替换并自动重启客户端
  • 可按 [S] 跳过本次、[D] 写入 .wmj_disable_auto_check 永久禁用
启动器层面（首次部署）
  • manifest 按 path / size / sha256 三元组寻址
  • 已匹配文件 零读盘（用 manifest.local.json 缓存短路）
  • 变更文件以 *.tmp.download 断点续传 → SHA 校验 → os.replace 原子替换
  • 最多 5 轮重试，抗偶发 SSL / 连接中断
  • 远端已移除的文件同步删除，但保护 launch.sh / launch.bat / manifest.local.json
环境变量
ROBOMASTER_SKIP_UPDATE=1 跳过更新直接启动赛场 / 离线
ROBOMASTER_UPDATE_URL自定义更新源基址默认 antientropy.xin
WMJ_DISABLE_AUTO_CHECK=1 客户端内不再弹更新仅保留启动器
手动操作
强制全量重装删除 manifest.local.json 后再运行
清理本地直接删除 RoboMasterClient/ 目录
性能 单次增量（如 Assembly-CSharp.dll 变更）通常 < 5 秒；首次全量约 300 MB； 命中缓存时零网络、零拷贝。

问题排查

实战沉淀的常见问题与解法。

Q1: 启动时提示缺少 UnityPlayer.so
解决 删除 manifest.local.json 后再次运行启动器。
Q2: Linux 提示 GLIBC_2.34 not found
解决 升级到 Ubuntu 22.04+ / Debian 12 / Fedora 37+。
Q3: 下载中断 SSL_ERROR_SYSCALL
解决 再次运行启动器即可，已完成的文件会被 SHA-256 跳过。
Q4: 比赛断网能启动吗
可以 首次安装完成后，manifest 拉取失败也会 fallback 启动；或设置 ROBOMASTER_SKIP_UPDATE=1。
Q5: 客户端内更新提示能关闭吗
能 弹窗中按 [D]，或设置 WMJ_DISABLE_AUTO_CHECK=1。
